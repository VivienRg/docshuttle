Document Portal Publishing Documents

Purpose & Scope

What this procedure covers and who it applies to

This procedure describes how to publish documents to your organization's Document Portal. The portal provides a high-performance, secure interface for accessing critical documents without navigating Google Drive directly. Viewers never need a Google account — access is controlled through Cloudflare Zero Trust instead.

Global Cache: The portal uses a persistent cache to ensure instant loading for all users. After adding or moving files in Google Drive, a manual Sync must be triggered to update the portal for the entire team. Simply refreshing your browser is not enough.
Scope

Applies to all team members authorized to contribute to the shared Google Drive folders linked to the Document Portal. Read-only users do not need to follow this procedure — they simply open the portal and navigate the document tree.

Prerequisites

Required access and supported formats before you begin

Required Access
  • Access to the organization's shared Google Drive
  • Contributor permission (or higher) on the source folders designated as portal roots
  • Portal access granted via Cloudflare Zero Trust (contact your system administrator if you cannot log in)
Supported File Formats
Google Docs
Google Sheets
Google Slides
PDF (.pdf)
Markdown (.md)
Plain Text (.txt)
HTML (.html)
Word (.docx)
Excel (.xlsx)
Files in unsupported formats will open via the Google Drive preview viewer rather than rendering inline. For the best experience, prefer the formats listed above.

Procedure

Follow these three steps each time you publish a new document

1
Organize in Google Drive
Place your file in the correct monitored folder
  • 1Upload your file directly to its designated folder in the shared Google Drive, or drag and drop it into the correct location.
  • 2If the document is stored elsewhere, use Google Drive's Add shortcut feature (Right-click the file → Organise → Add shortcut to Drive) to place a link into one of the monitored portal folders. The portal resolves shortcuts automatically — no copy is needed.
  • 3Ensure the filename is clear and descriptive. The filename (without extension) is the title that appears in the portal sidebar.
Tip: Create one shared root folder and have authors in other teams add shortcuts from their own folders. This keeps Drive organized while the portal surfaces everything in one place.
2
Trigger a Global Refresh
Sync the portal index for all users

Because the portal uses a persistent global cache, new files do not appear automatically when the page is refreshed. A manual sync is required after uploading to Drive.

  • 1Open the Document Portal in your browser.
  • 2In the left sidebar, locate the Refresh button next to the REPOSITORY header.
  • 3A confirmation modal will appear. Read the prompt and click Refresh Now to proceed.
The refresh process recursively scans all configured folders and subfolders. It may take several seconds depending on the depth of your folder structure. Once complete, the cache is updated globally — all users see the new content immediately.
3
Verify the Sync
Confirm your document appears correctly in the portal
  • 1Check the footer at the bottom of the sidebar. The Last Refresh timestamp should reflect the current date and time.
  • 2Locate your document in the nested folder tree. If it is inside a new folder, click the arrow ▶ next to the folder name to expand it.
  • 3Click the document to open it in the viewer and confirm it displays correctly.

Viewing & Interaction

Document modes, toolbar actions, and shareable links

Markdown Document Modes

For .md files, the portal provides two viewing modes accessible from the top toolbar:

Rendered Default
Formatted view with GitHub-style typography, code blocks, and headings. Recommended for reading.
Raw
Displays the unprocessed Markdown source in a monospace font. For technical review or copy-paste.
Open in Drive

To edit a document or leave comments, click Open in Drive in the top-right toolbar. This opens the source file in a new tab in Google Drive.

Shareable Permalinks

Every document has a stable /view/<id> URL shown in your browser's address bar when a document is open. Share this URL with colleagues to link them directly to a specific document — they will land on that document immediately after signing in.

Updating & Removing Content

When a Global Refresh is required and when it is not

Action in Google Drive Global Refresh Required? Notes
Edit content inside an existing document No Content is fetched from Drive on each open. Changes are live immediately.
Upload a new file Yes New files are not visible until the index is refreshed.
Rename a file or folder Yes The sidebar tree is rebuilt from the index on refresh.
Move a file to a different folder Yes The file's position in the sidebar changes after refresh.
Add a shortcut to a monitored folder Yes Shortcuts are resolved and indexed on refresh.
Delete a file or shortcut Yes Move to Trash in Drive, then trigger a Global Refresh to remove it from the portal.

Troubleshooting

Common issues and how to resolve them

Symptom Likely Cause Resolution
New file not appearing in sidebar Global cache still shows the previous state Click Refresh in the sidebar and confirm the modal. Wait for the spinner to complete.
"Last Refresh" shows an old timestamp No sync has been triggered since the last file change Manually trigger a refresh. The updated timestamp confirms the index was rebuilt for all users.
Word or Excel file fails to open Browser blocked the Drive preview, or you are not signed into the correct Google account Ensure you are signed into the Google account that has access to the Drive. Try opening in an incognito window.
PDF shows a "Sandbox" or blank error Browser extension conflict or strict security settings The portal adjusts security settings for PDFs automatically. Try a hard refresh (Ctrl+Shift+R / Cmd+Shift+R) or disable extensions temporarily.
Document opens but content looks wrong Cached version in portal KV is stale Trigger a Global Refresh. The portal re-fetches and re-caches the file content on next open.
Portal shows a "Root" folder only Configuration or credential issue (admin only) Contact your system administrator. This indicates a Drive API connectivity problem, not a user-level issue.

Roles & Responsibilities

Who is responsible for each part of the process

Role Responsibilities
Contributor Ensure files are correctly named and placed in the appropriate Drive folder. Trigger a Global Refresh after adding, renaming, moving, or deleting content. Verify the sync was successful.
Reader / Viewer Access and read documents via the portal. Report any sync discrepancies or missing documents to a Contributor or System Administrator.
System Administrator Maintain the Cloudflare Worker deployment and KV cache configuration. Manage Google OAuth credentials and folder ID configuration. Handle infrastructure-level issues (credentials, access policies, Drive API errors).
Document Control — Review annually or upon system changes. Distribute to all authorized contributors.
